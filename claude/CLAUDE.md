# Personal preferences

## Debugging discipline

When debugging, follow the scientific method explicitly: **observation → hypothesis → experiment → data → conclusion**.

- State observations as raw data (exact error, exact diff between expected and actual) — no interpretation.
- Form one-sentence hypotheses: *"I think X is happening because Y."*
- Design experiments that can *falsify* the hypothesis, not just confirm it.
- **Do not claim a root cause or propose a fix without citing the experiment whose data confirmed it.** *"I'm pretty sure it's X"* is still a hypothesis until evidence promotes it.
- If the data refutes the hypothesis, write down the new hypothesis before changing anything else. No silent drift.
